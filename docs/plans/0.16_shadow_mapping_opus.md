# Plan: Minimal directional-light shadow mapping

File: `docs/plans/0.16_shadow_mapping_opus.md`

Status: IMPLEMENTED (Milestone B). Used as the base plan, with two ideas adopted
from `0.16_shadow_mapping_glm.md`: a depth→color debug-copy pass so ImGui can
display the (depth-format) shadow map, and a shader-side reversed-Z epsilon for
bias instead of hardware depth bias. Fixed orthographic bounds and a 1024×1024
map were kept for v1. Deferred (not implemented): scene-AABB frustum fitting,
PCF, shadow-pass face culling, hardware depth bias, cascades.

## Goal

Add the **smallest** useful directional-light shadow mapping: one shadow map,
one directional light, orthographic light projection, a depth-only shadow pass,
a shadow lookup in the main pass, and an ImGui visualization of the map.

First success criterion: **a cube casts a visible shadow onto the ground plane
from the existing directional sun.**

## Non-goals

No cascaded shadow maps, render graphs, ECS, scene graphs, material systems,
generic lighting frameworks, multiple light types, future-proof abstraction
layers, or large renderer redesigns. This is a learning sandbox; favor simple,
readable code over engine architecture.

## Current render flow (baseline)

`Application::onRender()` (application.cpp:517):

1. Build ImGui frame (records UI draw data only).
2. Construct a fresh `RenderContext`; push camera (`view`, `projection`), light
   (`direction`, `ambient`, `diffuse`), sky.
3. `Scene::buildRenderCommands(ctx, selected)` appends a flat
   `std::vector<RenderCommand>` (ground + meshes + overlays).
4. `renderer_->draw(ctx, clearColor, uiCallback)`.

The renderer consumes only `RenderContext` (matrices + command list + light/sky
structs); it never sees Game/Scene types. This boundary is preserved.

`MetalRenderer::draw()` — one command buffer, three phases:

- **Scene pass** → offscreen `sceneColorTexture` + `depthTexture`. Reversed-Z:
  `clearDepth = 0.0`, depth compare `Greater`, write on, single
  `depthStencilState`. Sky drawn first (`Always`, write off); then the command
  list: cube/sphere/plane via `meshPipeline` (basic.metal), ground via
  `gridPipeline` (grid.metal).
- **Blit** `sceneColorTexture` → `drawable.texture`.
- **UI pass** ImGui onto the drawable (`loadAction = Load`).

Pipelines target `BGRA8Unorm` + `Depth32Float`. Mesh uniforms are a single
`ShaderUniforms {model, view, proj, normalMatrix}` at vertex buffer index 1;
fragment gets `Color`@0, `Light`@1.

Lighting convention: `LightSettings.direction` is the direction light *travels*;
surface-to-sun is `-direction`; the sky uses `sunDir = -light.direction`. The
shadow light-camera must be consistent with this.

## Integration plan

- **Generation:** new depth-only **shadow pass** in `MetalRenderer::draw()`,
  before the scene pass, sharing the command buffer. Renders caster meshes
  (cube/sphere/plane only — not ground or sky) from the light's viewpoint into a
  depth texture. No color attachment.
- **Sampling:** in the fragment shaders of **both** `basic.metal` (meshes
  receive / self-shadow) and `grid.metal` (ground receives — mandatory for the
  headline result). Each fragment transforms its world position by the light's
  view-projection, projects into the shadow map, and compares depth.
- **Light matrices:** computed on the **CPU** in `Application::onRender()` from
  `light.direction`. A **fixed orthographic box** centered on the world origin —
  no camera-frustum fitting (that path leads to CSM). `lightView = lookAt(eye =
  -dir * distance, center = origin, up)`, `lightProj = orthographic(...)`,
  `lightViewProj = lightProj * lightView`. Stored on `RenderContext` via
  `setShadow(const Mat4& lightViewProj)`; the renderer reads it for both the
  shadow-pass vertex uniforms and the main-pass fragment transform.

## Reversed-Z considerations

- **One convention.** Make the orthographic light projection **also reversed-Z**
  (`near→1, far→0`) so the shadow pass reuses the existing `Greater`/write-on
  depth state and `clearDepth = 0.0` unchanged. Mixing a standard `[0,1]` ortho
  with the reversed depth state silently breaks the map. (Reversed-Z gives no
  precision benefit for ortho — depth is linear — we do it only for consistency.)
- **Compare direction.** Reversed-Z ⇒ closer-to-light = larger depth. Fragment is
  **lit** when `currentDepth >= storedDepth - bias`; **in shadow** when
  `currentDepth < storedDepth - bias`. Copying a `[0,1]` tutorial gets this
  backwards (everything-in-shadow or inverted).
- **Bias sign is inverted** vs. textbook because the convention is flipped.
- **Metal specifics:** clip-space Z is `[0,1]` (target `1..0`); texture origin is
  top-left, so `uv = ndc.xy * 0.5 + 0.5` then **flip Y**; depth-only pass has no
  color attachment (color formats `Invalid`, set `depthAttachmentPixelFormat`);
  texture needs `usage = RenderTarget | ShaderRead`, private storage; prefer
  `[encoder setDepthBias:slopeScale:clamp:]` (native slope-scaled bias); when the
  sun is near-vertical (default `{0,-1,0}`), pick a non-parallel `up` (e.g.
  `{0,0,1}`) so `lookAt` doesn't degenerate.

## Milestones (independently buildable + observable)

**A — Light camera + depth pass + visualization (debug tool first).**
- Build: compiles. New ortho math, `setShadow`, shadow texture/pipeline/pass, and
  the ImGui `Image` panel.
- Visual: main render unchanged; the ImGui panel shows the scene from the sun's
  viewpoint as a depth image.
- Verify: panel shows the cube silhouette; rotating the sun changes it. (Building
  the visualizer first makes it the verification tool for the matrices — catches
  Y-flip / convention errors before they hide inside the lookup.)

**B — Shadow lookup → headline result.**
- Build: compiles. `basic.metal` + `grid.metal` sample the map; `lightViewProj` +
  texture + sampler bound in mesh and ground draws.
- Visual: **a cube casts a visible shadow onto the ground** under the existing
  sun. ✅
- Verify: screenshot via the existing capture path; shadow sits opposite the sun
  and tracks the sun direction.

**C — Quality knobs.**
- Build: compiles. Slope/constant bias + enable toggle wired to ImGui.
- Visual: acne / peter-panning tunable to clean; toggle removes the shadow.
- Verify: visual + screenshot regression at a known sun angle.

## Simplicity review (self-critique)

- `shadow.metal` is borderline — could reuse `vertex_main` with `lightViewProj`
  as view·proj. Kept separate for readability. (Decision below.)
- No `ShadowParams` struct yet; a single `bias` float (or hardcoded
  `setDepthBias`) suffices through Milestone B.
- No PCF / sampler-compare yet — plain depth sample + manual compare (hard edges)
  first; PCF is a one-line sampler change later.
- Fixed ortho box, no frustum fitting (avoids the CSM slope). Box covers the
  ~±50 ground extent; expose its size as a constant.
- Ground is a **receiver only**, never a caster (flat → no useful occlusion, only
  bias headaches).
- Visualization caveat: sampling `Depth32Float` in ImGui's default shader renders
  reversed-Z depth as a near-white blob (near-light ≈ 1.0). Good enough to
  verify; do not build colorizing infrastructure for it first.

## Decisions to confirm before implementation

1. Separate `shadow.metal` vs. reusing `vertex_main` for the depth pass.
2. Shadow map resolution — default **1024×1024**.
3. Bias via Metal's `setDepthBias` (encoder-level, native) vs. in-shader constant
   — leaning `setDepthBias`.
