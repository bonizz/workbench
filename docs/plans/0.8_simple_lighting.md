# Milestone 0.8 Plan — Simple Lighting

Status: APPROVED.

## Goal

Add simple ambient + directional diffuse lighting so primitive shapes show
visible depth. The cube should have differently bright faces, the sphere should
look round, and the plane should light consistently. `MeshRenderer.color` stays
the per-instance base color.

## Non-goals

- No shadows.
- No point lights.
- No materials system.
- No PBR.
- No textures.
- No UI-heavy lighting editor.
- No serialization changes.
- No new component type.

---

## 1. Current state

- `Vertex` is `{ float position[3]; float color[4]; }` (28 bytes). The color is
  unused; per-instance color is already a fragment uniform.
- `renderer/mesh_geometry.{h,cpp}` generates cube, sphere, and plane vertices
  with white color.
- `assets/shaders/basic.metal` draws flat color via `fragment_main`.
- `RenderContext` carries view/projection matrices and a list of `RenderCommand`s.
- `MetalRenderer` draws mesh shapes from static vertex/index buffers.
- The grid shader (`grid.metal`) is independent and should stay unchanged.

## 2. Design decisions

| # | Decision | Choice |
|---|----------|--------|
| D1 | Vertex layout | Replace the unused `color[4]` with `normal[3]`. `Vertex` becomes `{ float position[3]; float normal[3]; }` (24 bytes). This is the exact path anticipated by the 0.7 plan. |
| D2 | Normal generation | CPU-side in `mesh_geometry.cpp`: sphere `normal = normalize(pos)`; plane `normal = (0,1,0)`; cube duplicates vertices per face so each face has a constant face normal. |
| D3 | Normal transformation | Pass a `normalMatrix` (inverse-transpose of the model matrix upper 3×3) in `ShaderUniforms`. World-space normal = `normalMatrix * localNormal`. Must work correctly for non-uniform scale. |
| D4 | Light model | One global directional light: `ambient` + `max(dot(normal, lightDir), 0) * diffuse`. Color is `MeshRenderer.color` modulated by lighting. |
| D5 | Light ownership | `RenderContext` owns a small `LightSettings` struct (direction, ambient, diffuse). `Application` sets defaults once per frame. The renderer binds the same light data for every mesh draw. |
| D6 | Grid shader | Unchanged. Lighting uniforms are only bound for mesh draws. |
| D7 | Serialization | Unchanged. Lighting values are code-level constants for now, not saved in scene files. |
| D8 | Component model | Unchanged. No new component type. |

## 3. Vertex and geometry changes

### `src/engine/renderer/render_types.h`

```cpp
struct Vertex
{
    float position[3];
    float normal[3];
};

static_assert(sizeof(Vertex) == 24, "Vertex size mismatch");
static_assert(alignof(Vertex) == 4, "Vertex alignment mismatch");
```

Update the Metal `Vertex` struct in `basic.metal` to match:

```metal
struct Vertex
{
    packed_float3 position;
    packed_float3 normal;
};
```

### `src/engine/renderer/mesh_geometry.cpp`

- `makeSphere`: for each vertex, `normal = normalize(pos)`.
- `makePlane`: `normal = (0, 1, 0)` for all four vertices.
- `makeCube`: generate 24 vertices (4 per face × 6 faces) with face normals.
  Winding stays CCW. Indices stay `uint16_t` and 36 entries.

Helper for a lit vertex:

```cpp
Vertex litVertex(const Vec3& pos, const Vec3& normal)
{
    return Vertex{{pos.x, pos.y, pos.z}, {normal.x, normal.y, normal.z}};
}
```

Cube face layout example (front face, normal +Z):

```cpp
// +Z face
Vec3 n = {0, 0, 1};
data.vertices.push_back(litVertex({-1, -1,  1}, n));
data.vertices.push_back(litVertex({ 1, -1,  1}, n));
data.vertices.push_back(litVertex({ 1,  1,  1}, n));
data.vertices.push_back(litVertex({-1,  1,  1}, n));
```

Repeat for −Z, +X, −X, +Y, −Y faces, each with its own 4 vertices and 6 indices.

## 4. Math helper

### `src/engine/core/math.h` / `.cpp`

Add:

```cpp
using Mat3 = simd::float3x3;

// Inverse-transpose of the upper 3x3 of a 4x4 matrix, for transforming normals.
Mat3 normalMatrix(const Mat4& m);
```

Implementation: extract `float3x3` from the upper-left of `m`, invert it, then
transpose. Use `simd_inverse` if available from `<simd/simd.h>`; otherwise add a
small explicit 3×3 inverse helper. The result is passed to the GPU as a
`float3x3`.

## 5. Shader changes

### `assets/shaders/basic.metal`

```metal
struct Vertex
{
    packed_float3 position;
    packed_float3 normal;
};

struct Uniforms
{
    float4x4 modelMatrix;
    float4x4 viewMatrix;
    float4x4 projectionMatrix;
    float3x3 normalMatrix;
};

struct VertexOut
{
    float4 position [[position]];
    float3 worldNormal;
};

struct LightSettings
{
    float3 direction; // already normalized
    float ambient;
    float diffuse;
};

vertex VertexOut vertex_main(const device Vertex* vertices [[buffer(0)]],
                             constant Uniforms& uniforms [[buffer(1)]],
                             uint vertexID [[vertex_id]])
{
    Vertex in = vertices[vertexID];
    float4 worldPos = uniforms.modelMatrix * float4(in.position, 1.0f);
    VertexOut out;
    out.position = uniforms.projectionMatrix * uniforms.viewMatrix * worldPos;
    out.worldNormal = uniforms.normalMatrix * in.normal;
    return out;
}

fragment float4 fragment_main(VertexOut in [[stage_in]],
                              constant float4& uColor [[buffer(0)]],
                              constant LightSettings& light [[buffer(1)]])
{
    float3 n = normalize(in.worldNormal);
    float3 l = normalize(light.direction);
    float lambert = max(dot(n, l), 0.0f);
    float lit = light.ambient + light.diffuse * lambert;
    return float4(uColor.rgb * lit, uColor.a);
}
```

Color stays a fragment uniform at buffer 0; light data is a second fragment
buffer at index 1.

## 6. Renderer-side changes

### `src/engine/renderer/render_types.h`

- Update `ShaderUniforms` to include `simd::float3x3 normalMatrix`. The
  `static_assert(sizeof == 192)` will need to be updated (or removed if size
  changes).
- Add:

```cpp
struct LightSettings
{
    simd::float3 direction;
    float ambient;
    float diffuse;
};
```

- Extend:

```cpp
enum class MeshFragmentBufferIndex : uint32_t
{
    Color = 0,
    Light = 1,
};
```

### `src/engine/renderer/render_context.h` / `.cpp`

Add global light state:

```cpp
class RenderContext
{
public:
    // ... existing view/projection/command methods ...

    void setLight(const Vec3& direction, float ambient, float diffuse);
    const LightSettings& light() const;

private:
    LightSettings light_{{0.0f, -1.0f, 0.0f}, 0.3f, 0.7f}; // sensible default
};
```

`setLight` normalizes the direction on the CPU.

### `src/engine/scene/scene.cpp`

`buildRenderCommands` is unchanged except the renderer will now use the light
from `RenderContext` automatically.

### `src/engine/core/application.cpp`

Before calling `renderer.draw(...)`, set the default light:

```cpp
renderContext.setLight({0.0f, -1.0f, -0.5f}, 0.3f, 0.7f);
```

This is a single global light direction chosen to show depth clearly on cube
faces and spheres.

### `src/engine/renderer/metal_renderer.mm`

In the mesh draw path:

1. Compute `normalMatrix` from `cmd.transform` via `math::normalMatrix`.
2. Fill `ShaderUniforms` with `modelMatrix`, `viewMatrix`, `projectionMatrix`, and
   `normalMatrix`.
3. Bind color at `MeshFragmentBufferIndex::Color`.
4. Bind `rc.light()` at `MeshFragmentBufferIndex::Light`.

Static vertex/index buffers are unchanged because `mesh_geometry` now produces
normals in the same vertex layout.

## 7. Tests

### Unit tests in `src/tests/main.cpp`

All geometry tests are Metal-free because `mesh_geometry.cpp` is plain C++.

- **Vertex size**: assert `sizeof(Vertex) == 24` and `alignof(Vertex) == 4`.
- **Normals exist and are unit length** for every vertex of every shape.
- **Sphere normals**: for every vertex, `normalize(position) ≈ normal`.
- **Plane normals**: every normal equals `(0, 1, 0)`.
- **Cube normals**:
  - 24 vertices, 36 indices.
  - Each face's 4 vertices share the same normal.
  - Normals point in ±X, ±Y, ±Z directions only.
  - Face normals are outward-facing (e.g. +Z face normal is `(0,0,1)`).
- **Normal matrix helper**:
  - Rotation + uniform scale: normals rotate correctly and stay unit length.
  - Non-uniform scale (e.g. scale `(2, 1, 0.5)`): build a model matrix, transform
    a set of test normals with `normalMatrix()`, then normalize and verify they
    remain correct relative to the analytically expected inverse-transpose.
    Specifically, for a non-uniformly scaled cube face normal, the transformed
    normal should still be perpendicular to the transformed tangent in world
    space.
- **Existing tests remain green**: serialization round-trip, `assert.mesh`,
  `component.set_mesh`, DebugState, command discovery.

### `.wbs` tests

No new `.wbs` tests are required for lighting itself because the visual result
is shader-driven and not deterministically assertable without screenshots.
Existing `.wbs` tests must still pass unchanged.

### Build/test commands

- `./build.sh`
- `./test.sh`
- `./build/debug/sandbox --run-tests --exit`

### Manual smoke tests

1. Run `./build/debug/sandbox`.
2. Create cube, sphere, and plane objects with different colors.
3. Verify:
   - Cube faces have visibly different brightness.
   - Sphere looks round (not a flat circle).
   - Plane lights consistently across its surface.
   - Colors still work and are independent per object.
4. Run `render.capture lit_smoke.png` and visually inspect the lit image.

## 8. File-level implementation plan

**Modified engine files:**
- `src/engine/renderer/render_types.h` — new `Vertex` layout, `LightSettings`,
  updated `ShaderUniforms`, extended `MeshFragmentBufferIndex`.
- `src/engine/renderer/mesh_geometry.cpp` — generate normals for cube, sphere,
  plane.
- `src/engine/core/math.h` / `.cpp` — add `Mat3` alias and `normalMatrix()`.
- `src/engine/renderer/render_context.h` / `.cpp` — add `setLight()` / `light()`.
- `src/engine/renderer/metal_renderer.mm` — compute and bind normal matrix and
  light data.
- `src/engine/core/application.cpp` — set default light once per frame.
- `assets/shaders/basic.metal` — transform normals and compute ambient + diffuse.
- `src/tests/main.cpp` — normal/geometry unit tests.

**No changes to:**
- `MeshRenderer`, `RotateComponent`, `Component`, `GameObject`, `Transform`,
  scene hierarchy, serialization, editor, agent commands, grid shader,
  capture/bundle, window/platform.
