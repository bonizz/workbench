# Milestone 0.13 — Sky & Light Polish

## Goal
Make the procedural sky / directional light from 0.12 feel intuitive: sane
sun-size units, one-click sky presets, robust settings persistence, and a
documented light-direction convention. No new rendering capability.

## Current state (verified — reuse, don't rebuild)
- **Sun math is already angle-space.** `assets/shaders/sky.metal:56` computes
  `ang = acos(dot(rayDir, sunDir))` and thresholds with `sunSize` directly, so
  `sunSize` is *already* an angular radius in radians. **No shader edit needed**
  for scope item 1 — only the default + the editor slider are wrong-feeling.
- **Persistence already exists.** `Settings::loadLighting` / `saveLighting`
  (`core/settings.cpp:180-238`) read/write a `"lighting"` block, and
  `application.cpp` already calls them on load/save. The gap is *backward
  compatibility*: `loadLighting` reads every field inside one `try`, so a single
  missing field throws → `catch` → returns false → **all** out-params discarded.
  Item 3 is hardening this to per-field fallback, not new code.
- **Editor Lighting panel** `Editor::drawLightingPanel()` (`editor/editor.mm:384`)
  already owns both `lightSettings_` and `skySettings_` pointers and has the Sky
  `CollapsingHeader`. Presets (item 2) attach here; both pointers are in hand.
- **Settings test pattern** lives in `test_core.cpp:110+` (temp-path via
  `setSettingsPath`, save → load → assert). New round-trip tests go there.
- **Renderer state tests** live in `test_renderer.cpp` (plain C++). Preset-helper
  tests go there. Both files are in the explicit `tests` target list
  (`premake5.lua:101-105`).

## Scope & design (smallest path)

### 1. Sun-size UX — CPU/editor only, no shader change
- `render_types.h:71`: default `sunSize = 0.04f` → **`0.01f`** (~0.57° radius;
  in the requested 0.008–0.012 band).
- `editor.mm:426`: change the slider to the angular-radius range and make small
  values controllable:
  ```cpp
  ImGui::SliderFloat("Sun Radius (rad)", &skySettings_->sunSize,
                     0.001f, 0.05f, "%.4f", ImGuiSliderFlags_Logarithmic);
  ```
  Relabel from "Sun Size" → "Sun Radius (rad)" so the unit is explicit; log scale
  + `%.4f` gives usable resolution at the small end. Optional one-line
  `ImGui::SetItemTooltip` noting it's the sun's angular radius in radians.
- Leave `sunIntensity` slider (0–5) as-is.

### 2. Sky presets — new plain-C++ helper (testable)
New file pair `renderer/sky_presets.{h,cpp}` (kept out of `.mm` so it unit-tests):
```cpp
struct SkyPreset {
    const char* name;
    LightSettings light;
    SkySettings   sky;
};
// Fixed, ordered list. Returned by value/span; no globals.
const std::vector<SkyPreset>& skyPresets();   // Noon, Warm Sunset, Dusk / Blue Hour
```
Each preset sets sky colors + sunSize + sunIntensity **and** light
direction/ambient/diffuse (warm low-angle sun for sunset, dim blue for dusk).
Concrete starting values (tunable by eye later; tests assert the *helper returns
them*, not that they look good):
- **Noon** — current defaults; light dir `{-0.3,-1,-0.2}`, diffuse ~1.0,
  sunSize 0.01, intensity 1.0.
- **Warm Sunset** — orange horizon, violet zenith, warm sun; light dir near
  horizon `{-0.9,-0.15,-0.3}`, diffuse ~0.9, sunSize ~0.02, intensity ~1.5.
- **Dusk / Blue Hour** — deep-blue horizon/zenith, cool dim sun; low light,
  diffuse ~0.4, ambient up slightly, sunSize ~0.015, intensity ~0.6.

Editor: in the Sky header, a row of `ImGui::Button` per preset; on click,
`*skySettings_ = p.sky; *lightSettings_ = p.light;`. Guarded by the existing
`skySettings_ && lightSettings_` (both already non-null in the panel).

Register `sky_presets.cpp/.h` in the `tests` target file list (`premake5.lua`,
near `render_context.cpp`); the `sandbox` target already globs `renderer/**`.

### 3. Persist light + sky — backward-compat hardening
Rework `Settings::loadLighting` so missing fields fall back to the struct default
**in place** rather than discarding the whole block. Read field-by-field with
presence guards (mutate the incoming `light`/`sky`, which already hold defaults),
e.g. helper lambdas reading `obj.value("ambient", light.ambient)` and a
`readVec3(obj, key, fallback)` for the float3 colors/direction. Keep the outer
`try/catch` only for type-mismatch safety. `saveLighting` is unchanged. Do **not**
touch scene serialization. (Persistence wiring in `application.cpp` already
exists and needs no change.)

### 4. Documentation
- `docs/conventions.md` §4 (Math): add a short "Directional light convention"
  note — `LightSettings.direction` is the **light travel direction**; the
  surface-to-light vector and the visual sun direction are both `-direction`
  (`basic.metal` and `sky.metal` both negate). Call out the prior sign bug
  (mesh lit the face *away* from the sun) as the pitfall this convention prevents.
- This plan file documents the milestone decisions.

## Tests
- `test_core.cpp`: `Settings::saveLighting` → `loadLighting` round-trip for
  non-default `LightSettings` + `SkySettings` (assert every field). Plus a
  missing-fields test: write a partial `"lighting"` JSON (e.g. only
  `sky.sunSize`), load into a defaulted struct, assert present field applied and
  absent fields kept their defaults.
- `test_renderer.cpp`: `skyPresets()` returns the 3 expected names in order, and
  spot-check a couple of values per preset (e.g. Dusk diffuse < Noon diffuse,
  Sunset sun is warm). Assert the new `SkySettings` default `sunSize == 0.01f`.
- Existing tests stay green (no existing signatures change).

## Non-goals (restated)
No shadows, Hosek/Wilkie, clouds, HDR, env/IBL, new `LightComponent`, scene
serialization, or screenshot validation.

## Token-efficiency notes
- **Item 1 needs no shader edit** — the costly-to-reason-about Metal path is
  already correct; I won't reopen it beyond confirming the default.
- **Item 3 is an edit, not a rewrite** — persistence already exists; I touch only
  `loadLighting`'s body.
- New code is one small `.h/.cpp` pair (~40 lines) + ~15 lines of editor buttons
  + targeted test additions. No large-file regeneration.
- Visual confirmation of preset *look* is **manual by the user** (per non-goals);
  I verify via `./build.sh` + `./test.sh` only.
