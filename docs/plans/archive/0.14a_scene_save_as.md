# 0.14a — Scene "Save As"

Status: **implemented.** Follow-up to
[0.14 Scene Persistence](0.14_scene_persistence.md), which deferred the Open and
Save As modals. Open (a scene-list modal + Recent Scenes submenu) shipped later
via `SceneIO::listScenes`, `Settings::recentScenes`, and
`Application::loadScene`.

Shipped: `SceneIO` helper (`scene/scene_io.{h,cpp}`), `Application::saveSceneAs`
+ `consumeSaveAsRequest` with Save routing unsaved scenes to a request flag, the
Editor Save As modal (opened/drawn from the top-level `drawUI` path), and
`resolveScenePath`/`exists` tests in `test_serialization.cpp`.

Goal: let the user save the current scene to a **new** named file, and make
**Save** on an untitled/unsaved scene do the right thing instead of silently
no-op'ing.

## 0. Why this is needed now

`Application::saveScene()` (application.cpp:214) early-returns when
`loadedScenePath()` is empty:

```cpp
if (!scene_ || scene_->loadedScenePath().empty()) {
    // Save As is not implemented yet.
    return;
}
```

So a brand-new scene (`New`, or first launch with no `lastScene`) cannot be
saved at all from the UI, and `Ctrl+S` does nothing. Save As fixes both: it is
the only path that can *assign* a path, and Save falls back to it when unsaved.

## 1. Existing plumbing to reuse (don't rebuild)

- **IO**: `SceneSerializer::save(scene, path, error)` already creates parent
  directories and writes the versioned JSON. Save As needs no new IO code.
- **Path convention**: scenes live in `assets/scenes/`, extension
  `.scene.json`. `command.cpp` has a private `scenePath(filename)` =
  `"assets/scenes/" + filename`.
- **Title + dirty**: `updateWindowTitle()` and `markSceneDirty()` /
  `sceneDirty_` already exist and are wired through `Editor::markSceneDirty()`.
- **lastScene**: `Settings::saveLastScene(path)` (gated on `liveSimulation_` in
  `saveScene()`), and `loadedScenePath_` on `Scene`.
- **Editor↔App link**: `Editor` holds `Application* application_`
  (`setApplication`). Menu items already call `application_->newScene()` /
  `saveScene()`.
- **Text input**: Editor already uses fixed char buffers + `ImGui::InputText`
  (`commandBuffer_`, `scriptBuffer_`, `screenshotBuffer_`). **No `BeginPopupModal`
  exists yet** — Save As introduces the first modal in the editor.
- **Shortcuts**: `onKeyEvent` maps `Cmd/Ctrl+S → saveScene`, `+N → newScene`
  (application.cpp:459).

## 2. Ownership / boundary

Keep the 0.14 boundary intact:

| Piece | Owner |
|---|---|
| The modal (filename field, buttons, inline error/warning) | `Editor` (UI only) |
| Path resolution + serialize + dirty/title/lastScene update | `Application` (orchestration) |
| `assets/scenes` + `.scene.json` normalization | a tiny `SceneIO` helper (or `Application` private) |

Editor collects a **bare filename string** and calls
`Application::saveSceneAs(filename)`. Editor never touches the filesystem or
the serializer. This mirrors the §3 ownership table in 0.14.

## 3. New API

```cpp
// application.h  (public, called by Editor)
// Resolves `filename` under assets/scenes (adds .scene.json if absent),
// serializes the current scene there, and on success adopts it as the
// loaded path (so subsequent Save targets the new file). Returns false and
// fills `error` on failure; the modal stays open to show it.
bool saveSceneAs(const std::string& filename, std::string& error);
```

`saveScene()` changes so an unsaved scene routes to Save As instead of no-op.
Because Save lives in two trigger sites (menu + Ctrl+S) and the modal is owned by
the Editor, `Application::saveScene()` cannot itself *open* the modal cleanly.
Resolve with a small request flag the Editor polls:

```cpp
// application.h
bool consumeSaveAsRequest();   // returns true once if a Save-As was requested
private:
  bool saveAsRequested_ = false;
```

`saveScene()` becomes:

```cpp
void Application::saveScene() {
    if (!scene_) return;
    if (scene_->loadedScenePath().empty()) {
        saveAsRequested_ = true;   // Editor opens the modal next frame
        return;
    }
    std::string error;
    if (SceneSerializer::save(*scene_, scene_->loadedScenePath(), error)) {
        sceneDirty_ = false;
        if (liveSimulation_) Settings::saveLastScene(scene_->loadedScenePath());
        updateWindowTitle();
    }
    // else: surface error (status line / log) — see Risks
}
```

`saveSceneAs`:

```cpp
bool Application::saveSceneAs(const std::string& filename, std::string& error) {
    if (!scene_) { error = "No scene"; return false; }
    std::string path = SceneIO::resolveScenePath(filename, error);
    if (path.empty()) return false;                 // empty/invalid name
    if (!SceneSerializer::save(*scene_, path, error)) return false;
    scene_->setLoadedScenePath(path);
    sceneDirty_ = false;
    if (liveSimulation_) Settings::saveLastScene(path);
    updateWindowTitle();
    return true;
}
```

### Path normalization (`SceneIO::resolveScenePath`)
- Trim whitespace. Empty → fail with "Filename required".
- Reject path separators and `..` (keep saves inside `assets/scenes/`; no
  directory traversal). Reject empty-after-strip.
- If it does **not** already end in `.scene.json`, append `.scene.json`
  (so `arena` → `arena.scene.json`, `arena.scene.json` → unchanged).
- Return `"assets/scenes/" + name`. Directory creation is handled by
  `SceneSerializer::save`.

(Extract the existing `command.cpp::scenePath` into this shared helper so the
agent `scene.save` command and the UI agree on resolution.)

## 4. Editor modal

State on `Editor`:

```cpp
char  saveAsBuffer_[256] = {};
std::string saveAsError_;
```

Trigger (menu) — add to the existing Scene menu:

```cpp
if (ImGui::MenuItem("Save", "Ctrl+S"))    application_->saveScene();
if (ImGui::MenuItem("Save As...")) requestSaveAs();   // prefill + OpenPopup
```

`requestSaveAs()` prefills the buffer with a suggestion (current scene basename
without extension, else "untitled"), clears `saveAsError_`, and calls
`ImGui::OpenPopup("Save Scene As")`.

Each frame, in `drawUI` (top-level, always reached so the popup can render), and
poll the App's Ctrl+S-on-untitled request:

```cpp
if (application_ && application_->consumeSaveAsRequest()) requestSaveAs();

if (ImGui::BeginPopupModal("Save Scene As", nullptr, ImGuiWindowFlags_AlwaysAutoResize)) {
    ImGui::TextUnformatted("Save to assets/scenes/");
    bool submit = ImGui::InputText("##saveas", saveAsBuffer_, sizeof(saveAsBuffer_),
                                   ImGuiInputTextFlags_EnterReturnsTrue);
    ImGui::SameLine(); ImGui::TextDisabled(".scene.json");

    // Live overwrite hint (non-blocking).
    if (SceneIO::exists(saveAsBuffer_)) ImGui::TextColored(kWarn, "Will overwrite existing file.");
    if (!saveAsError_.empty())          ImGui::TextColored(kError, "%s", saveAsError_.c_str());

    bool empty = saveAsBuffer_[0] == '\0';
    ImGui::BeginDisabled(empty);
    if (ImGui::Button("Save") || submit) {
        if (application_->saveSceneAs(saveAsBuffer_, saveAsError_)) ImGui::CloseCurrentPopup();
        // else: keep open, error now shown
    }
    ImGui::EndDisabled();
    ImGui::SameLine();
    if (ImGui::Button("Cancel")) ImGui::CloseCurrentPopup();
    ImGui::EndPopup();
}
```

Notes:
- **Overwrite** is allowed but flagged inline (no second confirm dialog for v1).
  Simple and robust; a confirm step can be layered later.
- Enter submits, Esc/Cancel closes (imgui modals close on Esc by default).
- On failure the modal stays open with `saveAsError_` shown.

## 5. Flows

**Save As (explicit)**: menu → `requestSaveAs()` → modal → `saveSceneAs` →
serialize → adopt path → dirty cleared, title updated, lastScene updated.

**Save on unsaved scene**: menu/Ctrl+S → `saveScene()` sees empty path → sets
`saveAsRequested_` → Editor opens modal next frame → same as above.

**Save on saved scene**: writes in place (unchanged from today).

## 7. Tests

`SceneIO::resolveScenePath` is the testable core (the modal isn't):
- `"arena"` → `assets/scenes/arena.scene.json`.
- `"arena.scene.json"` → unchanged path.
- `""` / whitespace → fail.
- `"../evil"`, `"a/b"` → rejected (no traversal/subdirs).
- `saveSceneAs` round-trips: save untitled scene → file exists, `loadedScenePath`
  updated, `sceneDirty()` false; reload yields equal scene (reuse existing
  round-trip helpers in `test_serialization.cpp`).
- `saveScene` on empty path sets the request flag and writes nothing.

## 8. Risks / simplifications

- **First modal in the editor.** `BeginPopupModal` must be called every frame at
  a stable ID-stack location (top of `drawUI`), not nested inside the menu. The
  sketch above follows the standard imgui pattern.
- **Save error reporting for in-place Save** (not Save As) currently has nowhere
  to surface — today it silently fails. Out of scope here; note it. Minimal fix:
  log to the agent console. Recommend deferring.
- **Overwrite without confirm** is intentional for v1 (inline warning only).
- **No native file browser** — bare filename + fixed `assets/scenes/` dir, as in
  0.14. Open (a file *picker*) remains deferred; `SceneIO::listScenes()` is the
  natural seam to add it later.
- **Path safety**: reject separators/`..` so the UI can't write outside
  `assets/scenes/`.
