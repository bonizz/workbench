# Milestone 0.12 — Simple Procedural Sky

## Goal
Replace the flat clear-color background with a procedural sky: a horizon→zenith
gradient plus a sun disk, drawn behind all geometry. Sun direction reuses the
existing directional light. Editor controls for colors, sun size, sun intensity.
No Hosek/Wilkie, clouds, HDR, env lighting, or serialization.

## What already exists (reuse, don't rebuild)
- **Single offscreen scene pass** in `renderer/metal_renderer.mm:275` — clears
  color+depth, draws all meshes/ground, then blits to the drawable, then UI.
  Reverse-Z: `clearDepth = 0.0`, depth compare `Greater`
  (`metal_renderer.mm:148,284`).
- **Pipeline loader** `loadPipeline(...)` (`metal_renderer.mm:53`) — loads a
  `.metal` file by path, picks named vertex/fragment fns. Add the sky pipeline
  the same way the `grid`/`basic` pipelines are added (`metal_renderer.mm:131`).
- **`RenderContext`** carries per-frame global state to the renderer: camera
  matrices + `LightSettings` (`renderer/render_context.h`). Sky settings belong
  here too (global, like light) — not as a `RenderCommand`.
- **`LightSettings`** in `renderer/render_types.h:48` — the exact pattern to
  copy for `SkySettings`: `alignas(16)`, `simd::float3` fields, `static_assert`
  on size, mirrored 1:1 by a struct in the shader.
- **Light direction**: `Application::lightSettings_` (`application.h:121`),
  pushed each frame via `ctx.setLight(...)` (`application.cpp:331`). The sun
  direction is derived from it — no new direction field.
- **Editor Lighting panel** `Editor::drawLightingPanel()`
  (`editor/editor.mm:384`) owns a `LightSettings*` set from `Application`
  (`application.cpp:66`). Sky controls attach here (see §5).
- **Math**: `inverseMatrix(Mat4)` (`core/math.h:41`) for the inverse
  view-projection used to reconstruct world-space ray directions.

## Design decisions (smallest path)

### Background reconstruction = fullscreen triangle, no new pass, no new texture
Draw the sky as a 3-vertex fullscreen triangle **first**, inside the existing
scene pass, before any geometry. It covers the whole framebuffer so the prior
`clearColor` is no longer visible (clear still runs — harmless; leave
`clearColor_` as-is). Geometry then draws on top normally.

- Vertex shader generates the triangle from `vertex_id` (no vertex buffer):
  positions `(-1,-1)/(3,-1)/(-1,3)` in NDC.
- Sky must **not** occlude geometry and must **not** be occluded by the cleared
  depth. Use a dedicated depth-stencil state: `depthWriteEnabled = NO`,
  `compareFunction = Always`. Draw sky with it, then `setDepthStencilState` back
  to the existing geometry state before the command loop. (One new
  `id<MTLDepthStencilState> skyDepthState`.)

### Per-pixel world ray = unproject with inverse(view·projection)
Fragment needs the world-space view ray to evaluate the gradient (ray.y) and the
sun (dot(ray, sunDir)).
- CPU passes `invViewProj = inverseMatrix(projection * view)` and `cameraPos`
  (extracted from `inverseMatrix(view)` column 3) as fragment uniforms.
- Fragment: `float4 wp = invViewProj * float4(ndc.xy, 1.0, 1.0); wp /= wp.w;`
  → `rayDir = normalize(wp.xyz - cameraPos)`. Vertex shader forwards `ndc.xy`.

### Sun direction reuses the directional light
Renderer computes `sunDir = normalize(-rc.light().direction)` (light direction
is travel direction; the sun sits opposite) and passes it as a fragment uniform.
`SkySettings` holds **no** direction — satisfies "reuse current directional
light direction."

## Implementation

### 1. `SkySettings` — `renderer/render_types.h`
Mirror the `LightSettings` pattern:
```cpp
struct alignas(16) SkySettings
{
    simd::float3 horizonColor = {0.55f, 0.65f, 0.78f};
    simd::float3 zenithColor  = {0.12f, 0.22f, 0.45f};
    simd::float3 sunColor     = {1.0f, 0.95f, 0.85f};
    float sunSize      = 0.04f;  // angular radius-ish, 0..~0.3
    float sunIntensity = 1.0f;   // 0..~5
};
static_assert(sizeof(SkySettings) == 64, "SkySettings size mismatch");
```
(`simd::float3` is 16-byte sized/aligned: 16+16+16 + 4 + 4 → pad to 64.)
Add a `SkyFragmentBufferIndex` enum (Sky=0, InvViewProj=1, CameraPos=2,
SunDir=3) next to the existing index enums.

### 2. `RenderContext` — `renderer/render_context.{h,cpp}`
Add, mirroring `light_`:
```cpp
void setSky(const SkySettings& sky);
const SkySettings& sky() const { return sky_; }
// private: SkySettings sky_;
```

### 3. New shader — `assets/shaders/sky.metal`
- `struct SkySettings` matching the C++ layout (`float3` fields).
- `vertex SkyOut sky_vertex(uint vid [[vertex_id]])` → emits the 3 clip-space
  verts, forwards `ndc.xy`.
- `fragment float4 sky_fragment(...)`:
  - reconstruct `rayDir` (above);
  - `t = clamp(rayDir.y, 0, 1)` → `mix(horizonColor, zenithColor, t)` (smoothstep
    for a softer band);
  - sun: `d = dot(rayDir, sunDir);` disk = `smoothstep(1 - sunSize, 1 - sunSize*0.5, d)`
    plus a soft glow term; `color += sunColor * disk * sunIntensity;`
  - return `float4(color, 1.0)`.

### 4. Renderer wiring — `renderer/metal_renderer.mm`
- Add `id<MTLRenderPipelineState> skyPipeline` + `id<MTLDepthStencilState>
  skyDepthState` to `Impl`; release both in the dtor.
- In the ctor: `skyPipeline = loadPipeline(..., "assets/shaders/sky.metal",
  "sky_vertex", "sky_fragment", BGRA8Unorm, Depth32Float, false)`; build
  `skyDepthState` (Always / write NO).
- At the **start** of the scene encoder (before the command loop): if
  `skyPipeline`, set it + `skyDepthState`, compute `invViewProj`, `cameraPos`,
  `sunDir`, push as fragment bytes + `SkySettings`, then
  `drawPrimitives:MTLPrimitiveTypeTriangle vertexStart:0 vertexCount:3`. Then
  `setDepthStencilState:depthStencilState` to restore geometry state.

### 5. Editor controls — `editor/editor.{h,mm}`
To avoid touching the 4 window-state save/load sites for a new panel toggle,
**fold sky controls into the existing Lighting panel** under
`ImGui::CollapsingHeader("Sky")` (deliberate simplification — a standalone Sky
window would need `showSky_` plumbed through `drawUI`, the menu list, and
`loadWindowStates`/`saveWindowStates`).
- `editor.h`: add `SkySettings* skySettings_ = nullptr;` +
  `void setSkySettings(SkySettings* s) { skySettings_ = s; }`.
- `drawLightingPanel()`: after the light sliders, add the header with
  `ColorEdit3` for horizon/zenith/sun colors, `SliderFloat("Sun Size", …, 0.0f,
  0.3f)`, `SliderFloat("Sun Intensity", …, 0.0f, 5.0f)`. Guard on
  `skySettings_`.

### 6. Application wiring — `core/application.{h,cpp}`
- Add `SkySettings skySettings_;` member + accessors (mirror `lightSettings_`).
- In init (near `application.cpp:66`): `editor_->setSkySettings(&skySettings_);`
- In `onRender` (near `application.cpp:331`): `ctx.setSky(skySettings_);`

- Grid, meshes, selection highlight, gizmo still render correctly on top.
- Dragging the Lighting "Direction" moves the sun disk in the sky.
- Sky color / sun size / sun intensity sliders take visible effect.
